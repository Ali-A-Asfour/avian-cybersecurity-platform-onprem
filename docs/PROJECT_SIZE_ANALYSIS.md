## What Makes It So Big?

### 1. `.next/` Directory (868MB)
This is Next.js's build output and cache:
- Compiled pages and components
- Static assets
- Server-side rendering cache
- Webpack build artifacts

**Solution**: Exclude from zip. Regenerate with `npm run build`

### 2. `node_modules/` Directory (698MB)
NPM dependencies for the project:
- 1000+ packages
- TypeScript definitions
- Build tools and utilities

**Solution**: Exclude from zip. Reinstall with `npm install`

### 3. TypeScript Build Info (1.4MB)
Incremental compilation cache for faster rebuilds.

**Solution**: Exclude from zip. Regenerated automatically.

## How to Create Clean Transfer Zip

### Use the provided script:
```bash
./create-source-zip.sh
```

This creates a zip with only source files (~15-20MB).

## What Gets Excluded

❌ **Build Artifacts** (869MB)
- `.next/` - Next.js build
- `.swc/` - SWC compiler cache
- `*.tsbuildinfo` - TypeScript cache

❌ **Dependencies** (698MB)
- `node_modules/` - NPM packages

❌ **Environment Files**
- `.env.local` - Local secrets
- `.env.production` - Production secrets

❌ **Runtime Data**
- `storage/` - User uploads
- `.workflow-state/` - Session state
- `*.log` - Log files

## After Transfer Setup

1. Extract zip
2. Run `npm install` (downloads 698MB of dependencies)
3. Run `npm run build` (generates .next/ folder)
4. Copy `.env.example` to `.env.local` and configure
5. Run `npm run dev`

Total setup time: ~5 minutes (depending on internet speed)

## Recommendations

### For Repository Transfer
✅ Use `create-source-zip.sh` script
✅ Transfer only source files
✅ Include `.gitignore` to prevent future bloat
✅ Document setup steps

### For Git Repository
✅ Commit source files only
✅ Use `.gitignore` (already created)
✅ Let CI/CD handle builds
✅ Store secrets in environment variables

### For Deployment
✅ Build on deployment server
✅ Use Docker for consistent environments
✅ Cache `node_modules/` in CI/CD
✅ Use CDN for static assets
