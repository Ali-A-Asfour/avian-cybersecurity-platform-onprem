# Team Collaboration Guidelines

## Development Standards
- Always run tests before committing changes
- Use conventional commit messages (feat:, fix:, docs:, etc.)
- Update documentation when adding new features
- Follow the existing code structure and patterns

## Code Review Process
- All changes should be reviewed by at least one team member
- Use descriptive pull request titles and descriptions
- Link related issues or tickets in PR descriptions

## Kiro Usage Guidelines
- Use Kiro specs for complex feature development
- Document architectural decisions in the design.md files
- Keep task lists updated in specs/tasks.md files
- Use Kiro hooks for automated testing and validation

## Project Structure
- Follow the established folder structure in src/
- Keep API routes organized by feature domain
- Maintain consistent naming conventions
- Update type definitions when adding new data structures
